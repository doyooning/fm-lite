# 06. Next.js 화면 구성

## 라우팅 맵

| 경로 | 화면 | 렌더링 |
|---|---|---|
| `/` | 홈 | 서버 컴포넌트 |
| `/new-game` | 팀 선택 | 클라이언트 |
| `/game/[saveGameId]` | 게임 허브 | 서버 |
| `/game/[saveGameId]/squad` | 선수단 조회 | 서버 |
| `/game/[saveGameId]/opponent` | 상대 분석 | 서버 |
| `/game/[saveGameId]/tactics` | 전술 설정 | 클라이언트 |
| `/game/[saveGameId]/match/[matchId]` | 경기 진행 | 클라이언트 (핵심) |
| `/game/[saveGameId]/match/[matchId]/result` | 경기 결과 | 서버 |
| `/game/[saveGameId]/competition` | 대회 진행 현황 | 서버 |

## 화면별 구성

### 1. 홈 (`/`)
- 게임 타이틀 + [새 게임 시작] 버튼
- localStorage에 userId·saveGameId 있으면 [이어하기] 카드 (팀명, 현재 라운드)
- 최초 접속 시 `POST /users`로 익명 유저 생성

### 2. 팀 선택 (`/new-game`)
- 8팀 카드 그리드 (2×4). 카드: 팀명, 등급 배지(색상), 콘셉트 한 줄, 지역별 전력 미니 바
- 카드 클릭 → 우측/하단 패널에 상세(주요 선수 3명) + [이 팀으로 시작]
- 시작 → `POST /save-games` → `/game/[id]`로 이동

### 3. 게임 허브 (`/game/[id]`)
- 상단 바(게임 내 공통 layout): 내 팀명, 대회명, 현재 라운드
- 다음 경기 카드: 상대팀, 라운드, [상대 분석] [전술 설정] [경기 시작(전술 설정 후 활성화)]
- 미니 대진표 + [대회 현황 보기]
- 우승/탈락 상태면 결과 배너 표시

### 4. 선수단 조회 (`/squad`)
- 포지션별 그룹핑 테이블 (GK/DF/MF/FW)
- 행: 등번호, 이름, 포지션, 능력치 8종(StatBar — 수치+색상 바), 특성 칩(hover 시 효과 설명)
- 정렬: 포지션 → 종합치

### 5. 상대 분석 (`/opponent`)
- 상대 팀 헤더 (팀명, 등급)
- 지역별 전력 비교 차트 (내 팀 vs 상대 — 가로 막대 4쌍)
- 강점/약점 리스트, 키 플레이어 카드, 예상 전술 요약
- 하단 [전술 설정하러 가기]

### 6. 전술 설정 (`/tactics`)
- formation: 3개 선택 카드 (간단한 포지션 도트 표시)
- mentality / pressing / lineHeight: 세그먼트 컨트롤
- attackStyle: 4개 라디오 카드
- 선택 조합에 대한 간단한 힌트 텍스트 (예: "높은 압박은 체력 소모가 큽니다")
- [저장] → PUT 후 허브로 / [저장 후 경기 시작] → match 화면으로

### 7. 경기 진행 (`/match/[matchId]`) — 핵심 화면
```
┌─────────────────────────────────┐
│  RSF  1 : 0  GRF        58'     │  ← ScoreBoard + MinuteBar(진행 게이지)
├─────────────────────────────────┤
│  43' ⚽ GOAL! 김강찬의 침투!      │
│  51' 상대가 라인을 올립니다        │  ← EventFeed (아래로 쌓임, 자동 스크롤)
│  ...                            │
├─────────────────────────────────┤
│  ❓ 상대가 압박해 옵니다. 대응은?   │
│  [역습 노림] [점유 유지] [수비 강화]│  ← ChoiceModal (WAITING_CHOICE일 때)
└─────────────────────────────────┘
```
- 플로우: 진입 시 `POST /matches/{id}/start` → 응답 events를 **600~1000ms 간격으로 순차 표시** (setInterval) → 마지막이 CHOICE면 선택 UI 표시 → 선택 시 `POST /choices` → 반복 → `FINISHED`면 결과 페이지로 자동 이동
- 새로고침 복구: `GET /events`로 로그 재로딩, 상태가 WAITING_CHOICE면 선택 UI 복원
- 이벤트 타입별 아이콘/색 (⚽골, 🧤선방, 🔄전술변화, ❓선택지)

### 8. 경기 결과 (`/match/[matchId]/result`)
- 최종 스코어 (승부차기 있으면 병기), 승/패 배너
- 통계: 점유율, 슛/유효슛, 베스트 플레이어
- 주요 이벤트 타임라인 (골/선방만 필터)
- [대회 현황으로] — 다음 라운드 진출 또는 탈락 안내

### 9. 대회 진행 현황 (`/competition`)
- 8강 → 4강 → 결승 브래킷 뷰 (모바일: 세로 스택, 데스크톱: 가로 트리)
- 내 팀 경기 하이라이트 표시, 완료 경기 스코어, 우승 시 트로피 연출
- 다음 내 경기 있으면 [다음 경기 준비] 버튼

## 공통 사항

- 모바일 우선 반응형 (Tailwind 기본 브레이크포인트), 다크 테마 기본
- API 클라이언트: `lib/api/client.ts`에서 `NEXT_PUBLIC_API_BASE_URL` + `X-User-Id` 헤더 자동 첨부, 에러 토스트 공통 처리
- 화면 자체는 AI 도구로 빠르게 생성하되, **types/와 lib/api/의 타입·함수 시그니처를 먼저 고정**해 두면 생성 품질과 일관성이 크게 좋아짐
